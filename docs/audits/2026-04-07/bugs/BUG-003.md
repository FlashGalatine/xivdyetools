# BUG-003: `eslint-disable` for `@typescript-eslint/no-explicit-any` in OAuth Logger

- **Severity:** LOW
- **Category:** Type Safety
- **File:** `apps/oauth/src/middleware/logger.ts:87`

## Description

The OAuth logger middleware uses `// eslint-disable-next-line @typescript-eslint/no-explicit-any` to suppress a type safety warning. This allows untyped data to flow through the logger helper function.

## Impact

Minimal — the logger helper is internal middleware and the `any` type is used for the Hono context variable typing, which has known limitations with generic middleware patterns.

## Recommendation

Use a narrowed Context type or `unknown` instead of `any`:

```typescript
// Instead of: any
// Use: unknown or the specific Hono context type
```

This is a code quality nit, not a functional issue.

## Effort

MINIMAL — Single type annotation change.

## Resolution

**Status:** RESOLVED  
**Date:** 2026-04-07

Replaced all `Record<string, any>` and `Context<any>` types in worker-middleware with `Record<string, unknown>` and unparameterized `Context` (leveraging Hono's `ContextVariableMap` module augmentation). Removed all eslint-disable comments.
