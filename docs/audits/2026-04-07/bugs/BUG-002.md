# BUG-002: `console.error` for JSON Corruption Instead of Structured Logger

- **Severity:** LOW
- **Category:** Observability
- **File:** `apps/presets-api/src/services/preset-service.ts:62, 172, 204, 329, 354`

## Description

The BUG-012 fix (JSON.parse error handling) uses `console.error()` for logging corrupted preset rows:

```typescript
console.error(`[BUG-012] Preset ${row.id}: invalid JSON in 'previous_values', defaulting to null`);
console.error(`[BUG-012] Skipping corrupted preset row id=${row.id}:`, error);
```

While functionally correct, this bypasses the structured logger used throughout the rest of the codebase (`@xivdyetools/logger`). The structured logger provides:
- Request ID correlation
- Secret redaction
- Consistent format for log aggregation
- Severity levels

## Impact

Corrupted preset rows are logged without request context, making it harder to trace which user request encountered the corruption.

## Recommendation

Replace `console.error` with the request-scoped logger:

```typescript
logger.error('[BUG-012] Skipping corrupted preset row', { presetId: row.id, error });
```

## Effort

LOW — Replace ~5 instances of `console.error` with logger calls. Requires passing logger instance to `rowToPreset()` or using a module-level logger.
