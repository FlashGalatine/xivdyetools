# BUG-001: Disabled Strict TypeScript Checks in Worker Apps

- **Severity:** LOW
- **Category:** Type Safety / Technical Debt
- **Files:**
  - `apps/discord-worker/tsconfig.json:7-9`
  - `apps/presets-api/tsconfig.json:7-9`
  - `apps/oauth/tsconfig.json` (similar overrides)
  - `apps/moderation-worker/tsconfig.json` (similar overrides)

## Description

All worker apps override the strict base `tsconfig.base.json` to disable:

```json
{
  "noUnusedLocals": false,
  "noUnusedParameters": false,
  "noImplicitReturns": false
}
```

The base config at the monorepo root correctly enables these strict checks. Worker apps disable them, likely for development convenience.

## Impact

- Unused imports and variables accumulate without compiler warnings
- Missing return statements in functions go undetected
- Dead code is harder to identify

## Recommendation

Re-enable strict checks. For unused parameters, prefix with underscore (`_param`) which TypeScript recognizes as intentionally unused:

```typescript
// Before: app.use('*', async (c, next) => { ... })
// After: app.use('*', async (_c, next) => { ... })
```

## Effort

MEDIUM — Requires fixing all unused variable warnings across worker apps before re-enabling.

## Resolution

**Status:** RESOLVED  
**Date:** 2026-04-07

Strict TypeScript checks (`noUnusedLocals`, `noUnusedParameters`, `noImplicitReturns`) re-enabled for all 5 worker apps (api-worker, oauth, moderation-worker, discord-worker, presets-api). All unused variables/imports cleaned up, implicit return warnings fixed.
