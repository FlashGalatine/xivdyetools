# SEC-005: Placeholder Environment Variable in moderation-worker wrangler.toml

- **Severity:** LOW
- **Category:** Configuration
- **File:** `apps/moderation-worker/wrangler.toml:36`

## Description

The moderation-worker's `wrangler.toml` contains a placeholder value:

```toml
DISCORD_CLIENT_ID = "YOUR_MODERATION_BOT_CLIENT_ID"
```

If this placeholder is not overridden in the actual deployment environment, the worker will start with an invalid client ID. While the `validateEnv()` function at startup checks for required environment variables, the `DISCORD_CLIENT_ID` check verifies that the value is non-empty — which the placeholder satisfies.

## Risk

LOW — In practice, the bot would fail to authenticate with Discord using an invalid client ID, which would be immediately obvious. However, the worker would start successfully and potentially process requests before discovering the misconfiguration.

## Recommendation

Add a placeholder detection check to `validateEnv()`:

```typescript
if (env.DISCORD_CLIENT_ID.startsWith('YOUR_')) {
  errors.push('DISCORD_CLIENT_ID contains placeholder value');
}
```

Or use Cloudflare's secret management (`wrangler secret put`) for the client ID instead of `wrangler.toml`.

## Effort

MINIMAL — ~3 lines in env validation.
