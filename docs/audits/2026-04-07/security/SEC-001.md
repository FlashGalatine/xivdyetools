# SEC-001: moderation-worker Missing Global `onError` Handler

- **Severity:** MEDIUM
- **Category:** Information Disclosure
- **File:** `apps/moderation-worker/src/index.ts`
- **Status:** NEW

## Description

The moderation-worker Hono application does not define a global `app.onError()` handler. All other workers (discord-worker, presets-api, oauth) implement environment-aware error handlers that suppress stack traces in production and return generic error messages.

Without a global error handler, unhandled exceptions will produce Hono's default error response, which may include stack traces and internal error details.

## Risk

In production, if an unhandled exception occurs in the moderation-worker, the default Hono error response could leak:
- Internal file paths
- Stack traces revealing code structure
- Dependency names and versions

This information aids attackers in crafting targeted exploits.

## Comparison

Other workers implement this pattern:

```typescript
// presets-api/src/index.ts:231
app.onError((err, c) => {
  const isDev = c.env.ENVIRONMENT === 'development';
  return c.json({
    error: 'Internal Server Error',
    ...(isDev && { stack: err.stack }),
  }, 500);
});
```

The moderation-worker is missing this entirely.

## Recommendation

Add a global `onError` handler to the moderation-worker matching the pattern used in other workers:

```typescript
app.onError((err, c) => {
  const isDev = c.env.ENVIRONMENT === 'development';
  console.error('Unhandled error:', err.message, isDev ? err.stack : '');
  return c.json({
    error: 'Internal Server Error',
    ...(isDev && { message: err.message, stack: err.stack }),
  }, 500);
});
```

## Effort

LOW — ~10 lines of code, pattern already established in other workers.
