# SEC-002: `innerHTML` Usage in modal-container.ts

- **Severity:** LOW
- **Category:** Cross-Site Scripting (Potential)
- **File:** `apps/web-app/src/components/modal-container.ts:244`
- **Status:** **FIXED** (2026-04-07)

## Description

The `modal-container.ts` component set `content.innerHTML = modal.content` where `modal.content` was a string. While only set programmatically (not from user input), this pattern created a potential XSS vector if future changes introduced user-controlled content.

## Resolution

1. Removed `string` from `ModalConfig.content` type — now only accepts `HTMLElement`
2. Removed the `innerHTML` branch in `modal-container.ts`
3. Converted all string-based modal content to DOM elements:
   - `dye-action-dropdown.ts` — slot selection modal (was innerHTML template literal + setTimeout hack)
   - `result-card.ts` — slot selection modal (same pattern)
   - `preset-tool.ts` — delete confirmation dialog
4. Updated tests to use `HTMLElement` content

The `innerHTML` usage for the close button SVG icon (line 223) remains — this is a hardcoded SVG literal with no interpolation, not a security concern.

## Files Changed

- `apps/web-app/src/services/modal-service.ts` — `content` type narrowed to `HTMLElement`
- `apps/web-app/src/components/modal-container.ts` — removed innerHTML branch
- `apps/web-app/src/components/dye-action-dropdown.ts` — DOM elements instead of HTML string
- `apps/web-app/src/components/v4/result-card.ts` — DOM elements instead of HTML string
- `apps/web-app/src/components/v4/preset-tool.ts` — DOM element instead of string
