## Description

The discord-worker validates `Content-Length` before reading the body for webhook endpoints (10KB limit). However, the presets-api and oauth workers do not enforce a general request body size limit before JSON parsing.

## Current Mitigations

1. **Cloudflare Workers limit:** The platform enforces a 100MB request body limit
2. **Input validation:** The presets-api validates field lengths (name: 50 chars, description: 200 chars, etc.) after parsing
3. **Content-Type enforcement:** Only `application/json` is accepted for mutations

## Risk

LOW — While a very large JSON payload could consume memory during parsing, Cloudflare's platform limits and the Workers 128MB memory cap provide natural bounds. The validation layer catches oversized fields after parsing.

## Recommendation

Add a lightweight body size check middleware to presets-api and oauth:

```typescript
app.use('*', async (c, next) => {
  const contentLength = parseInt(c.req.header('content-length') || '0');
  if (contentLength > 100_000) { // 100KB limit
    return c.json({ error: 'Payload too large' }, 413);
  }
  await next();
});
```

## Effort

LOW — ~5 lines of middleware per worker.
