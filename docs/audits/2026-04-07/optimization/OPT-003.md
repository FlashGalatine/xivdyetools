# OPT-003: Multiple JSON.stringify/parse Cycles in Budget Pipeline

- **Impact:** LOW
- **Category:** CPU / Latency
- **File:** `apps/discord-worker/src/services/budget/index.ts`

## Description

The budget command pipeline processes dye pricing data through multiple transformation steps. Each step may serialize/deserialize data structures via `JSON.stringify()`/`JSON.parse()`, adding unnecessary CPU overhead.

This pattern is common in pipelines that were built incrementally, where each stage was developed independently.

## Impact

LOW — The budget command is a user-initiated action with expected latency. The additional CPU from redundant serialization is small compared to the Universalis API fetch time.

## Recommendation

If budget command latency becomes a concern:
1. Profile the pipeline to identify serialization hotspots
2. Pass typed objects between stages instead of re-serializing
3. Use `structuredClone()` where deep copies are needed (faster than JSON round-trip for complex objects)

## Effort

LOW — Refactoring is straightforward but the impact is minimal. Prioritize only if budget command latency is reported.
