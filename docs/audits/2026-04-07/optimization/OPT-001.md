# OPT-001: Full Collection Load from KV into Memory

- **Impact:** LOW
- **Category:** Memory
- **File:** `apps/discord-worker/src/services/user-storage.ts:227-241`

## Description

The `getCollections()` function loads all user collections from KV into memory in a single `get()` call:

```typescript
const collections = await this.kv.get(key, 'json');
```

With current limits (`MAX_COLLECTIONS = 50`, `MAX_DYES_PER_COLLECTION = 20`), the maximum data size is bounded (~50KB worst case). However, if these limits are increased in the future, loading all collections into memory could:
- Approach KV value size limits (25MB max)
- Consume significant Worker memory (128MB limit)
- Increase latency for KV reads

## Current Safety

The limits are enforced at write time, so the current implementation is safe. This is a fragility note, not a current bug.

## Recommendation

If collection limits are ever increased significantly:
1. Consider pagination or lazy loading
2. Use KV metadata for collection indexes (list without loading full data)
3. Migrate to D1 for relational queries instead of loading full JSON blobs

N/A — No immediate action needed. Document as a scaling concern.
