## Description

Per project memory ([`project_acquisition_strings_drift.md`](../../../../../C:/Users/DrawF/.claude/projects/c--Users-DrawF-OneDrive-Projects-CodingProjects-XIVProjects/memory/project_acquisition_strings_drift.md), 2026-04-28), `colors_xiv.json` was renamed: `'Crafting'` ŌåÆ `'The Firmament'` and `'Treasure Chest'` ŌåÆ `'Venture Coffers'`. The filter constants in [`DyeFilter.ts`](../../../../packages/core/src/services/dye/DyeFilter.ts) (`CRAFT_ACQUISITIONS`, etc.) were updated to match.

Eight test fixtures across four test files still construct mock dyes with the old string `acquisition: 'Crafting'`:

```typescript
// e.g., DyeDatabase.test.ts:42
const mockDye: Dye = {
  itemID: 5729,
  name: 'Snow White',
  acquisition: 'Crafting',  // ŌåÉ stale; production data now says 'The Firmament'
  // ...
};
```

## Impact

- **Today:** Tests pass because none of these fixtures invoke filter logic that compares against `CRAFT_ACQUISITIONS` (`['The Firmament', 'Venture Coffers']`).
- **Risk:** A new test that adds `excludeCraftDyes: true` to its filter input will silently NOT filter the mock ŌĆö because `'Crafting' !== 'The Firmament'` ŌĆö leading to a false-positive pass that hides a real filter regression.
- This is the same drift class as the production bug fixed in [`DyeFilter.ts`](../../../../packages/core/src/services/dye/DyeFilter.ts) on the same date; the fix landed in production code but missed the test fixtures.

## Recommendation

Replace `acquisition: 'Crafting'` with `acquisition: 'The Firmament'` in all 8 fixture instances. Also recommend a guard test that asserts every string in `CRAFT_ACQUISITIONS` and the `'Treasure Chest'` ŌåÆ `'Venture Coffers'` rename appears in the unique set of `acquisition` values from the real `colors_xiv.json`, to catch future renames automatically:

```typescript
// New test: packages/core/src/services/dye/__tests__/DyeFilter.contract.test.ts
import colorsData from '../../../data/colors_xiv.json';
import { CRAFT_ACQUISITIONS } from '../DyeFilter.js';

test('CRAFT_ACQUISITIONS values exist in colors_xiv.json', () => {
  const acquisitions = new Set(colorsData.map(d => d.acquisition));
  for (const a of CRAFT_ACQUISITIONS) {
    expect(acquisitions.has(a)).toBe(true);
  }
});
```
