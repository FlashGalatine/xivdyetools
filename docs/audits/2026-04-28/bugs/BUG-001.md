# BUG-001: api-worker uses bare `console.error` for unhandled errors

- **Severity:** MEDIUM
- **Category:** Observability / Logging Consistency
- **File:** [`apps/api-worker/src/index.ts:134`](../../../../apps/api-worker/src/index.ts#L134)

## Description

The api-worker's global error handler logs unhandled errors with bare `console.error`, breaking the structured logging pattern established in `@xivdyetools/worker-middleware`:

```typescript
// apps/api-worker/src/index.ts:115-145
app.onError((err, c) => {
  const requestId = getRequestId(c);
  const isDev = c.env.ENVIRONMENT === 'development';

  if (err instanceof ApiError) {
    return c.json({ ... }, err.statusCode as 400);
  }

  // Unexpected error — log and return generic message
  console.error(`[${requestId}] Unhandled error:`, isDev ? err : { name: err.name, message: err.message });

  return c.json({ ... }, 500);
});
```

The worker already imports `getRequestId` from `@xivdyetools/worker-middleware` at line 14 — meaning the structured logger is also a one-line import away. Sibling workers (`presets-api`, `discord-worker`) route unhandled errors through `getLogger(c)` which adds context (request ID, service name, structured fields) and integrates with downstream log aggregation.

## Impact

- Unhandled errors lose structured context (no service name, no log level discrimination, no JSON output)
- Inconsistent with the observability pattern shipped in 2026-04-07's REFACTOR-001 (worker-middleware extraction)
- If api-worker is wired to Cloudflare Logpush or external log shippers in the future, these `console.error` lines won't be parsed alongside structured logger output

## Recommendation

Replace the `console.error` with a structured logger call. The pattern used in [`apps/presets-api/src/index.ts`](../../../../apps/presets-api/src/index.ts) is:

```typescript
import { getLogger, requestIdMiddleware } from '@xivdyetools/worker-middleware';

app.onError((err, c) => {
  const logger = getLogger(c);
  const requestId = getRequestId(c);
  // ...
  logger?.error('Unhandled error', { requestId, name: err.name, message: err.message, stack: isDev ? err.stack : undefined });
  return c.json({ ... }, 500);
});
```

Also add `loggerMiddleware()` to the global middleware chain (currently only `requestIdMiddleware` is registered at line 31).

## Effort

LOW — single file change in [`apps/api-worker/src/index.ts`](../../../../apps/api-worker/src/index.ts), plus add `loggerMiddleware` to the chain. Existing logger configuration in `worker-middleware` should drop in unchanged.
