# [BUG-047]: findComplementaryPair never excludes the base dye (self-match on near-neutrals)

## Severity
LOW

## Type
Logic error / edge case

## Location
- `packages/core/src/services/dye/HarmonyGenerator.ts:85-110` (`findComplementaryPair`, no exclusions)
- Contrast: `packages/core/src/services/dye/HarmonyGenerator.ts:328-331` (`findHarmonyDyesByOffsets` seeds `usedDyeIds` with `baseDye.id`)

## Description
Every offset-based harmony method first resolves the base dye and excludes it from subsequent matches (`const usedDyeIds = new Set<number>([baseDye.id])` at `:331`). `findComplementaryPair` does neither — it inverts the hex and searches with no exclusions on all three of its paths (non-HSV color space at `:94`, deltaE at `:101` passing `new Set()`, default hue path at `:105`).

For near-neutral inputs the RGB inverse of a mid-gray is essentially the same color (`ColorManipulator.invert('#808080') === '#7F7F7F'`), so the returned "complementary" dye is the very dye the input color maps to — a self-pair presented as a complement.

## Reproduction Scenario
```ts
dyeService.findComplementaryPair('#7F7F80');
// → returns the same gray dye that dyeService.findClosestDye('#7F7F80') returns
```

## Evidence
```ts
// HarmonyGenerator.ts:97-105
const complementaryHex = ColorManipulator.invert(hex);

// Use DeltaE matching if requested
if (options?.algorithm === 'deltaE') {
  return this.findClosestByDeltaE(complementaryHex, new Set(), options);  // ← empty exclusion set
}

// Find closest dye, excluding Facewear
return this.findClosestNonFacewearDye(complementaryHex);                  // ← no excludeIds
```

## Suggested Fix
Resolve the base dye first and exclude it, mirroring `findHarmonyDyesByOffsets`:
```ts
const baseDye = this.search.findClosestDye(hex);
const exclude = baseDye ? [baseDye.id] : [];
...
if (options?.algorithm === 'deltaE') {
  return this.findClosestByDeltaE(complementaryHex, new Set(exclude), options);
}
return this.findClosestNonFacewearDye(complementaryHex, exclude);
```

## Why It's Hidden
Harmony tests use saturated colors where the RGB inverse is genuinely distant, so the base dye never wins the complementary search. Only desaturated/gray inputs — a realistic user case in FFXIV glamour palettes — hit the self-match.

## Status

**FIXED 2026-07-19** — `findComplementaryPair` resolves the base dye first and excludes it on all three match paths, mirroring `findHarmonyDyesByOffsets` — no more self-pairs on near-neutral inputs.
