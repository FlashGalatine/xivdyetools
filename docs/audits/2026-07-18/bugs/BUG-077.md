# [BUG-077]: Welcome modal contradicts "shows only once" and permanently suppresses the changelog auto-popup

## Severity
LOW

## Type
Behavioral/state-machine bug (documented intent vs. implementation)

## Location
- `apps/web-app/src/components/welcome-modal.ts:104` (`dontShowAgain = false` default)
- `apps/web-app/src/components/welcome-modal.ts:144-149` (`onClose` conditional mark)
- `apps/web-app/src/components/welcome-modal.ts:116-120` (`markAsSeen` is the only first-time writer of `LAST_VERSION_VIEWED`)
- Interplay: `apps/web-app/src/components/changelog-modal.ts:51-61` (`shouldShow` returns false when no last version)

## Description
The file header documents "Shows only once unless reset by user" (`:5`), but `onClose` only calls `markAsSeen()` when `dontShowAgain` is true. Closing via the X button, backdrop click, or Escape (all enabled, `:141-143`) leaves `WELCOME_SEEN` unset, so the welcome modal reappears on **every** visit.

Because `markAsSeen()` is also the only initial writer of `LAST_VERSION_VIEWED` (`:119`), such users additionally never satisfy `ChangelogModal.shouldShow()` — the "What's New" auto-popup is permanently suppressed for them across all future updates, while the welcome modal paradoxically keeps nagging.

## Reproduction Scenario
Fresh profile → welcome modal appears → press Escape → reload: welcome modal appears again. Deploy a new app version: no changelog popup for this user, welcome modal instead — indefinitely, until they click "Get Started"/"Take the Tour" or tick the checkbox.

## Evidence
```ts
// welcome-modal.ts:144-149
onClose: () => {
  if (this.dontShowAgain) {          // default false; X/backdrop/Escape leave it false
    WelcomeModal.markAsSeen();
  }
  this.modalId = null;
},

// welcome-modal.ts:116-120
static markAsSeen(): void {
  StorageService.setItem(STORAGE_KEYS.WELCOME_SEEN, true);
  // Also set version so changelog doesn't show for this version
  StorageService.setItem(STORAGE_KEYS.LAST_VERSION_VIEWED, APP_VERSION);
}

// changelog-modal.ts:54-57 — never true while LAST_VERSION_VIEWED is unset
if (!lastVersion) {
  return false;
}
```

## Suggested Fix
Decide the intent and encode it: either always call `markAsSeen()` in `onClose` (the checkbox then only controls a hypothetical "show again" preference), or at minimum write `LAST_VERSION_VIEWED` unconditionally on close so the changelog logic gains a baseline:

```ts
onClose: () => {
  if (this.dontShowAgain) WelcomeModal.markAsSeen();
  StorageService.setItem(STORAGE_KEYS.LAST_VERSION_VIEWED, APP_VERSION);
  this.modalId = null;
},
```

## Why It's Hidden
Tests close the modal through its buttons; real users routinely press Escape or click the backdrop. The downstream effect (missing changelog popups) surfaces only across version updates, far from the cause.

> Source: evidence/web-frontends-analysis.md (2026-07-18 deep-dive, web-frontends area)

## Status

**FIXED 2026-07-19** — chose the documented "shows only once" intent: any close (X/backdrop/Escape included) calls `markAsSeen()`, which also seeds `LAST_VERSION_VIEWED` so the changelog auto-popup is no longer permanently suppressed for those users.
