# [BUG-048]: sortByProperty claims "undefined sorted to end" but treats undefined as equal

## Severity
LOW

## Type
Documentation/behavior mismatch in exported utility

## Location
- `packages/core/src/utils/index.ts:492-506` (comparator; doc claim at `:492`)
- Exported publicly via `packages/core/src/index.ts:116`

## Description
The JSDoc for `sortByProperty` states "Handles undefined properties (sorted to end)", but the comparator returns `0` for any comparison involving `undefined`: both `aVal < bVal` and `aVal > bVal` are `false` when either side is `undefined`, so the comparison yields `0` ("equal"). Because the sort is stable, undefined-valued items simply keep their original positions *interleaved* with the rest — not moved to the end. Any consumer of this exported utility relying on the documented contract gets silently wrong ordering.

## Reproduction Scenario
```ts
sortByProperty([{ age: 30 }, { age: undefined }, { age: 20 }], 'age');
// documented:  [{age:20}, {age:30}, {age:undefined}]
// actual:      [{age:30}, {age:undefined}, {age:20}]  — original order preserved around undefined
```

## Evidence
```ts
// utils/index.ts:500-505
return [...array].sort((a, b) => {
  const aVal = a[property];
  const bVal = b[property];
  const comparison = aVal < bVal ? -1 : aVal > bVal ? 1 : 0;  // ← undefined vs X → 0 ("equal")
  return order === 'asc' ? comparison : -comparison;
});
```

## Suggested Fix
Either correct the doc, or implement the promise:
```ts
return [...array].sort((a, b) => {
  const aVal = a[property];
  const bVal = b[property];
  if (aVal === undefined) return bVal === undefined ? 0 : 1;  // undefined always last
  if (bVal === undefined) return -1;
  const comparison = aVal < bVal ? -1 : aVal > bVal ? 1 : 0;
  return order === 'asc' ? comparison : -comparison;
});
```
(Note: with the fix, undefined stays last regardless of `order` — document that choice.)

## Why It's Hidden
Tests sort arrays with fully-populated properties; mixed defined/undefined inputs are never exercised, and a "wrong order" result raises no exception.

## Status

**FIXED 2026-07-19** — `sortByProperty` implements the documented contract: undefined (and null) values sort to the end regardless of order; doc updated.
