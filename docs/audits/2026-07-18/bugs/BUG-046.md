# [BUG-046]: DefaultRateLimiter is not concurrency-safe — concurrent callers burst through

## Severity
LOW

## Type
Race condition / async anti-pattern

## Location
- `packages/core/src/services/APIService.ts:77-96` (`DefaultRateLimiter`)
- Usage: `packages/core/src/services/APIService.ts:596-597` (single fetch), `:663-664` (batch fetch)

## Description
`DefaultRateLimiter` splits "wait" and "record" into two non-atomic steps with no slot reservation between them. N concurrent callers (e.g., several `getPriceData` calls for *different* items — which do not deduplicate against each other) all read the same `lastRequestTime`, all compute the same sleep duration, and all fire simultaneously when it elapses. The intended 200 ms spacing (`API_RATE_LIMIT_DELAY`) degrades to a burst of N requests with zero spacing.

## Reproduction Scenario
```ts
// Cold cache:
await Promise.all([
  apiService.getPriceData(5729),
  apiService.getPriceData(5730),
  apiService.getPriceData(5731),
]);
// → three Universalis requests within the same millisecond instead of 200 ms apart
```

## Evidence
```ts
// APIService.ts:86-95
async waitIfNeeded(): Promise<void> {
  const timeSinceLastRequest = Date.now() - this.lastRequestTime;
  if (timeSinceLastRequest < this.minDelay) {
    await sleep(this.minDelay - timeSinceLastRequest);   // ← all concurrent callers sleep the same amount
  }
}

recordRequest(): void {
  this.lastRequestTime = Date.now();   // ← only updated after everyone already passed waitIfNeeded
}
```

## Suggested Fix
Reserve the slot synchronously before sleeping — synchronous state update means no interleaving is possible:
```ts
export class DefaultRateLimiter implements RateLimiter {
  private nextAvailable = 0;
  constructor(private minDelay: number = API_RATE_LIMIT_DELAY) {}

  async waitIfNeeded(): Promise<void> {
    const now = Date.now();
    const slot = Math.max(now, this.nextAvailable);
    this.nextAvailable = slot + this.minDelay;   // reserve BEFORE awaiting
    if (slot > now) await sleep(slot - now);
  }

  recordRequest(): void { /* no-op — folded into waitIfNeeded */ }
}
```

## Why It's Hidden
Rate limiting is disabled or mocked in tests (that is the documented purpose of the injectable `RateLimiter`). In production the burst only matters against the real upstream and surfaces as occasional, unattributed 429s.

## Status

**FIXED 2026-07-19** — `DefaultRateLimiter` reserves the next slot synchronously before awaiting; concurrent callers space out by minDelay (regression test added). `recordRequest` is a compatibility no-op.
