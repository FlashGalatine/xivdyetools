# [BUG-044]: PaletteService.samplePixels — division by zero at maxSamples=1; maxSamples never clamped

## Severity
LOW

## Type
Boundary condition / missing input validation

## Location
- `packages/core/src/services/PaletteService.ts:321-337` (sampler)
- Contrast: clamping of `colorCount` and `maxIterations` at `packages/core/src/services/PaletteService.ts:356-371`

## Description
`extractPalette` clamps `colorCount` (to [1, 10]) and `maxIterations` (to [1, 100]) with INPUT-003 warnings, but `maxSamples` is validated nowhere:

- `maxSamples: 1` with ≥2 pixels → the interpolation index is `(0 * (n-1)) / (1-1) = 0/0 = NaN` → `pixels[NaN]` is `undefined` → downstream `rgbDistance(undefined, …)` throws `TypeError: Cannot read properties of undefined (reading 'r')` deep inside k-means.
- `maxSamples: 0` or negative → the `pixels.length <= maxSamples` guard is false, the sampling loop body never runs → silently returns `[]` → `extractPalette` returns `[]` with no warning, masquerading as "no colors found".

## Reproduction Scenario
```ts
paletteService.extractPalette(pixels, { maxSamples: 1 }); // any image with ≥2 pixels → TypeError
paletteService.extractPalette(pixels, { maxSamples: 0 }); // silently returns []
```

## Evidence
```ts
// PaletteService.ts:331-334
for (let i = 0; i < maxSamples; i++) {
  const index = Math.round((i * (pixels.length - 1)) / (maxSamples - 1)); // ← ÷0 when maxSamples === 1
  samples.push(pixels[index]);
}
```

## Suggested Fix
Clamp like the sibling options, with the same INPUT-003-style warning:
```ts
if (opts.maxSamples < 2) {
  this.logger.warn(`PaletteService.extractPalette: maxSamples ${opts.maxSamples} clamped to minimum 2`);
}
const maxSamples = Math.max(2, opts.maxSamples);
```
(and/or special-case `maxSamples === 1` to return `[pixels[0]]`).

## Why It's Hidden
Tests use the default (10 000) or realistic values; nothing exercises the degenerate option values, and the `≤ 0` variant fails silently rather than crashing.
