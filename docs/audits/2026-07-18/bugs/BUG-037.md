# [BUG-037]: /stats unique-user count silently caps at 1000 — KV list() pagination ignored

## Severity
MEDIUM

## Type
Boundary condition / incorrect aggregation

## Location
`apps/discord-worker/src/services/analytics.ts:252` (stats-key list) and `:282-283` (unique-user count)

## Description
`KVNamespace.list()` returns at most 1000 keys per call; `list_complete` / `cursor` are never consulted. Once daily active users exceed 1000, `/stats` reports exactly 1000 unique users forever. The stats-counter list at `:252` has the same latent flaw (breaks past 1000 distinct `stats:*` keys — ~996 distinct command counter names; unlikely, but the code should not depend on it).

## Reproduction Scenario
1. Seed 1500 `usertrack:{today}:{userId}` keys (or reach >1000 DAU organically).
2. Call `getStats()` (backing `/stats`).
3. `uniqueUsersToday === 1000` regardless of the true count.

## Evidence
```ts
// apps/discord-worker/src/services/analytics.ts:282-283
const userListResult = await kv.list({ prefix: `${USER_TRACK_PREFIX}${today}:` });
const uniqueUsersToday = userListResult.keys.length;   // max 1000 per page; cursor ignored
```

## Suggested Fix
Loop with the cursor (values are not needed — only key counts):
```ts
let uniqueUsersToday = 0;
let cursor: string | undefined;
do {
  const page = await kv.list({ prefix: `${USER_TRACK_PREFIX}${today}:`, cursor });
  uniqueUsersToday += page.keys.length;
  cursor = page.list_complete ? undefined : page.cursor;
} while (cursor);
```
Alternative with O(1) reads: maintain a daily counter incremented inside `trackUniqueUser`'s "not seen yet" branch — its existing read-first check (`:205-208`) already deduplicates, so the increment fires exactly once per user per day.

## Why It's Hidden
Mock KV list implementations return everything unpaginated, so tests can never observe the cap. Production only crosses 1000 DAU under real load, and a flat-lined stat at a round number reads as a plateau, not a bug.

## Status

**FIXED 2026-07-19** — the unique-user count follows `kv.list` cursors, so `/stats` no longer caps at 1000 DAU.
