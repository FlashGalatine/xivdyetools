# [BUG-029]: universalis-proxy datacenter/world whitelist is stale — EU "Shadow" DC and its worlds rejected with 400

## Severity

MEDIUM

## Type

Data staleness / input validation false negative

## Location

`apps/universalis-proxy/src/config/datacenters.ts:19-140` (`VALID_DATACENTERS`, `VALID_WORLDS`); enforced at `apps/universalis-proxy/src/index.ts:162-164`

## Description

`VALID_DATACENTERS` lists 11 datacenters (ending at Materia) and `VALID_WORLDS` lists 84 worlds. The EU **Shadow** datacenter (added to the live game in 2024, worlds **Innocence, Pixie, Titania, Tycoon**) is absent from both sets. Legitimate requests such as `/api/v2/aggregated/Shadow/5729` or `/api/v2/aggregated/Innocence/5729` are rejected with `400 Invalid datacenter or world name`, even though upstream Universalis serves them — EU players on Shadow simply cannot use market-board features. Any future world/DC additions (Square Enix adds them roughly yearly) will silently break the same way.

Korean/Chinese datacenters are also absent — acceptable if intentional (the web-app does not target those regions), but this should be an explicit comment rather than an implicit omission.

## Reproduction Scenario

```
curl "https://proxy.xivdyetools.app/api/v2/aggregated/Innocence/5729"
# → 400 {"error":"Invalid datacenter or world name"}

curl "https://universalis.app/api/v2/aggregated/Innocence/5729"
# → 200 (valid world upstream)
```

## Evidence

```ts
// config/datacenters.ts:19-35
export const VALID_DATACENTERS = new Set([
  // Japan
  'elemental', 'gaia', 'mana', 'meteor',
  // North America
  'aether', 'crystal', 'dynamis', 'primal',
  // Europe
  'chaos', 'light',          // ← no 'shadow'
  // Oceania
  'materia',
]);
// VALID_WORLDS (:43-140) likewise lacks innocence / pixie / titania / tycoon
```

## Suggested Fix

Immediate:

```ts
// Europe
'chaos', 'light', 'shadow',
// EU - Shadow
'innocence', 'pixie', 'titania', 'tycoon',
```

Structural (self-healing): on whitelist miss, fall back to checking against the `/api/v2/data-centers` + `/worlds` payloads the worker already caches for 24 h (`CACHE_CONFIGS.dataCenters` / `.worlds`), keeping the static set as the fast path. That removes the recurring maintenance trap entirely.

## Why It's Hidden

The whitelist's tests were written from the same hardcoded list, so they can never detect drift; nothing cross-checks against the live `/data-centers` endpoint that this very worker proxies. The failure only affects players homed on the newest DC — a minority whose bug reports ("prices don't load for me") look like user-side issues.
