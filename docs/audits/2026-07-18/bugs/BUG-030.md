# [BUG-030]: api-worker `/v1/match/within-distance` applies excludeIds/filters after core `limit` truncation ‚Äî qualifying dyes silently dropped

## Severity

MEDIUM

## Type

Logic error / filter-limit ordering

## Location

- `apps/api-worker/src/routes/match.ts:113-131`
- Contributing constraint: `packages/core/src/services/dye/DyeSearch.ts:28-37` (`FindWithinDistanceOptions` has no `excludeIds` field)

## Description

The handler passes the client's `limit` (default 20) into `dyeService.findDyesWithinDistance()`, which truncates the distance-sorted result to `limit` **before** the handler removes `excludeIds` matches and applies vendor/craft/expensive/metallic filters. Excluded or filtered dyes therefore consume result slots: with `limit=20` and 30 dyes inside `maxDistance`, if 5 of the closest 20 are excluded the client receives 15 results (`resultCount: 15`) even though 25 dyes qualify ‚Äî silently, with no indication anything was dropped.

The sibling endpoint `/v1/match/closest` does **not** have this problem: it pushes `excludeIds` into `FindClosestOptions` (match.ts:52-56), so the two endpoints behave inconsistently for the same query parameters.

## Reproduction Scenario

1. `GET /v1/match/within-distance?hex=808080&maxDistance=50&limit=5` ‚Üí note the 5 returned itemIDs.
2. `GET /v1/match/within-distance?hex=808080&maxDistance=50&limit=5&excludeIds=<the 3 closest itemIDs>` ‚Üí returns **2** results, not 5 ‚Äî dyes ranked 6-8 (which qualify and are not excluded) never appear.

## Evidence

```ts
// match.ts:113-131
const options: FindWithinDistanceOptions = {
  maxDistance,
  limit,                       // ‚Üê core truncates to `limit` here
  matchingMethod: method,
  weights,
};
let dyes = dyeService.findDyesWithinDistance(hex, options);

if (excludeIdsRaw) {
  const excludeInternalIds = new Set(resolveExcludeIds(excludeIdsRaw));
  dyes = dyes.filter((d) => !excludeInternalIds.has(d.id));  // ‚Üê removes from already-truncated list
}
dyes = applyDyeFilters(dyes, filters);                        // ‚Üê further shrinks it
```

```ts
// packages/core/src/services/dye/DyeSearch.ts:28-37 ‚Äî no excludeIds option available
export interface FindWithinDistanceOptions {
  maxDistance: number;
  limit?: number;
  matchingMethod?: MatchingMethod;
  weights?: OklchWeights;
}
```

## Suggested Fix

Fetch unbounded (the database is only 136 entries), filter first, truncate last:

```ts
let dyes = dyeService.findDyesWithinDistance(hex, {
  maxDistance, matchingMethod: method, weights, limit: 136,
});
if (excludeIdsRaw) { /* filter */ }
dyes = applyDyeFilters(dyes, filters);
dyes = dyes.slice(0, limit);   // apply the client limit AFTER filtering
```

Optionally: add `excludeIds` to core's `FindWithinDistanceOptions` for parity with `FindClosestOptions`.

## Why It's Hidden

Tests assert "returns ‚â§ limit results" and "excluded IDs are absent" as separate properties; both pass. The defect only manifests when exclusions/filters overlap the *head* of the distance-sorted list ‚Äî an interaction no test constructs, and one that API consumers perceive as "fewer matches near this color than expected" rather than a bug.
