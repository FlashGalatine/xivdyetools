# [BUG-016]: Pagination parameters unvalidated — `page=abc` 500s; negative `limit` bypasses the 50 cap and dumps the table

## Severity
MEDIUM

## Type
Edge Case / Boundary Condition (input validation, resource amplification)

## Location
- File: `apps/presets-api/src/handlers/presets.ts`
- Line(s): 61-62
- Also: `apps/presets-api/src/services/preset-service.ts:176-188` (offset math and bind)
- Function: `presetsRouter.get('/')`, `getPresets`

## Description
`page ? parseInt(page, 10) : undefined` yields `NaN` for non-numeric input; `NaN` propagates into `offset = (page - 1) * limit` and `.bind(..., limit, offset)` — D1 rejects `NaN` bindings, surfacing as an unhandled 500. `limit` is capped only from above (`Math.min(parseInt(limit,10), 50)`): `?limit=-1` produces `LIMIT -1`, which SQLite treats as **no limit**, returning the entire presets table in one response (memory/egress amplification; combined with `search` this is an easy DoS lever). `?page=0` or negative pages produce negative offsets (SQLite treats them as 0 — mostly benign but inconsistent with the `has_more` math).

## Reproduction Scenario
1. `GET /api/v1/presets?page=abc` → 500 `INTERNAL_ERROR` (NaN bind rejected by D1).
2. `GET /api/v1/presets?limit=-1` → every approved preset in a single payload, bypassing the documented 50 cap.

## Evidence
```ts
// presets.ts:61-62
page: page ? parseInt(page, 10) : undefined,
limit: limit ? Math.min(parseInt(limit, 10), 50) : undefined, // Cap at 50 for performance
```
```ts
// preset-service.ts:176, 185-188
const offset = (page - 1) * limit;
... .bind(...params, limit, offset).all(...)
```

## Suggested Fix
Clamp both parameters at parse time:
```ts
const parsedPage = Math.max(1, Number.parseInt(page ?? '', 10) || 1);
const parsedLimit = Math.min(Math.max(1, Number.parseInt(limit ?? '', 10) || 20), 50);
```

## Why It's Hidden
Tests use well-formed integers, and the inline comment ("Cap at 50") reads as if bounds are fully handled. The `Math.min` cap silently passes both `NaN` (`Math.min(NaN, 50) === NaN`) and negatives (`Math.min(-1, 50) === -1`), neither of which any test sends.

> Source: evidence/d1-workers-analysis.md (2026-07-18 deep-dive, d1-workers area)

## Status

**FIXED 2026-07-18** — pagination params are now clamped in the GET /presets handler: non-numeric values default (page 1 / limit 20), page floors at 1, limit clamps to 1..50 (SQLite `LIMIT -1` can no longer dump the table). Regression tests added.
