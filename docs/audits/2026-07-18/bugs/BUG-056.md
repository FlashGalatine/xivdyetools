# [BUG-056]: Emoji in SVG text that the bundled fonts cannot render (tofu in generated PNGs)

## Severity
LOW

## Type
Rendering defect / runtime-portability assumption (font glyph coverage under resvg)

## Location
- `packages/svg/src/preset-swatch.ts:40-47` (`CATEGORY_DISPLAY` emoji icons), `:137-148` (icon prepended to the rendered title)
- `packages/svg/src/random-dyes-grid.ts:79` (default title `'рџЋІ Random Dyes'`)
- Contradicted by the package's own note: `packages/svg/src/budget-comparison.ts:421` вЂ” *"(no emoji/О” вЂ” fonts lack those glyphs in resvg)"*

## Description
The SVG package's output is rasterized by resvg with a fixed set of bundled fonts (Space Grotesk / Onest / Habibi / Noto Sans SC+KR subsets вЂ” `base.ts:290-299`). None of these contain emoji glyphs, which `budget-comparison.ts` explicitly documents and works around. Yet:

- `generatePresetSwatch` unconditionally renders `` `${categoryDisplay.icon} ${name}` `` into the card title, where `icon` is one of вљ”пёЏ рџЏ›пёЏ рџЌ‚ рџЋ‰ рџЋЁ рџЊђ.
- `generateRandomDyesGrid` defaults its title to `'рџЋІ Random Dyes'`.

These code points have no glyph in any bundled font, so the rendered PNG shows a tofu/missing-glyph box (or nothing) where the icon should be. This is live in production: `apps/discord-worker/src/handlers/commands/preset.ts:853` calls `generatePresetSwatch` with real category IDs.

## Reproduction Scenario
1. Run `/preset show <any preset>` on the Discord bot.
2. `generatePresetSwatch` emits `<text>рџЋЁ My Preset</text>`; discord-worker rasterizes via resvg with the bundled fonts.
3. The posted PNG's title begins with a missing-glyph box before the preset name.

## Suggested Fix
Keep `CATEGORY_DISPLAY` icons for Discord *message* text (they render fine there), but strip emoji from SVG text:

```ts
// preset-swatch.ts вЂ” drop the icon from the SVG title line:
text(width / 2, PADDING + 24, name, { ... })

// random-dyes-grid.ts вЂ” emoji-free default; callers pass localized titles anyway:
title = 'Random Dyes',
```
If icons in images are wanted long-term, replace them with small inline `<path>`/`<circle>` glyphs or a subset emoji font (weigh against the worker bundle-size budget).

## Why It's Hidden
SVG unit tests assert on the SVG *string* вЂ” the emoji is present and correctly XML-escaped, so tests pass. The glyph gap only appears after rasterization, which no test performs, and in Discord the broken box sits inside a decorative image title that's easy to overlook.
