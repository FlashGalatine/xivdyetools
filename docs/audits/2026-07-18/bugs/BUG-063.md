# [BUG-063]: `generateGradientColors(…, 1)` divides by zero and produces `#NaNNaNNaN`

## Severity
LOW

## Type
Edge case / division by zero (NaN propagation into output)

## Location
`packages/svg/src/gradient.ts:201-214` (`generateGradientColors`, division at 209); NaN formatting via `interpolateColor` at `:176-193`; exported publicly via `packages/svg/src/index.ts:39-43`

## Description
`generateGradientColors` computes `ratio = i / (stepCount - 1)`. With `stepCount === 1` the sole iteration evaluates `0 / 0 = NaN`; `interpolateColor` then does `Math.round(r1 + (r2 - r1) * NaN) = NaN`, and `NaN.toString(16)` yields the literal string `"NaN"` — the function returns `['#NaNNaNNaN']`, which callers inject verbatim into SVG `fill` attributes, producing an unstyled/black swatch in the rendered PNG. With `stepCount === 0` it returns `[]` (benign), and negative counts also return `[]`.

`generateGradientBar` protects itself with a `steps.length < 2` throw (gradient.ts:58-60), but `generateGradientColors` is exported independently from the package root with no guard, so any caller computing a step count from user input (e.g. a `/gradient steps:1` command option) hits this.

## Reproduction Scenario
```ts
import { generateGradientColors } from '@xivdyetools/svg';
generateGradientColors('#FF0000', '#0000FF', 1);
// → ['#NaNNaNNaN']
```
Any downstream `rect(x, y, w, h, '#NaNNaNNaN')` renders as an invalid-fill rectangle in resvg output.

## Evidence
```ts
// packages/svg/src/gradient.ts:201-214
export function generateGradientColors(
  startColor: string,
  endColor: string,
  stepCount: number
): string[] {
  const colors: string[] = [];

  for (let i = 0; i < stepCount; i++) {
    const ratio = i / (stepCount - 1);        // stepCount=1 → 0/0 → NaN
    colors.push(interpolateColor(startColor, endColor, ratio));
  }

  return colors;
}
```

## Suggested Fix
```ts
export function generateGradientColors(
  startColor: string,
  endColor: string,
  stepCount: number
): string[] {
  if (stepCount < 2) {
    return stepCount === 1 ? [startColor] : [];   // or throw, matching generateGradientBar
  }
  ...
}
```

## Why It's Hidden
Bot command validation clamps step counts to sensible ranges (e.g. 2–10) before this function is reached, so the edge never fires in current flows, and unit tests use ≥2 steps. Because the failure mode is a *string* (`"#NaNNaNNaN"`) rather than an exception, nothing throws — the bad value flows silently into SVG markup and only manifests as a subtly wrong image if a future caller forgets the external clamp.

## Status

**FIXED 2026-07-19** — `generateGradientColors` guards `stepCount < 2` (returns `[startColor]` for 1, `[]` otherwise), eliminating the 0/0 → `#NaNNaNNaN` path.
