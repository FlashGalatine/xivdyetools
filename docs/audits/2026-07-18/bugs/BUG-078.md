# [BUG-078]: Tutorial prompt can fire for the previous tool after navigating away

## Severity
LOW

## Type
Stale-closure / uncancelled timer race

## Location
- `apps/web-app/src/components/v4-layout.ts:111-118` (`promptTutorialIfFirstVisit`)
- Invoked at `apps/web-app/src/components/v4-layout.ts:461-465` (after tool load)

## Description
The tutorial first-visit prompt is scheduled with a raw 800 ms `setTimeout` that is never stored or cleared, and it captures the tool at schedule time. If the user visits Harmony for the first time and clicks over to Budget within 800 ms, the *Harmony* tutorial prompt pops while the Budget tool is on screen; accepting it starts a tutorial that spotlights selectors that do not exist in the current DOM.

Additionally, `markTutorialOffered(tool)` runs **before** the delay (`:112`), so the one-time offer is burned even if the prompt is then suppressed inside the timeout window by the modal-open or tutorial-active checks — the user permanently loses that tool's first-visit prompt.

## Reproduction Scenario
Fresh profile with welcome modal already dismissed. Navigate to `/harmony` (first visit), then within 800 ms click "Budget" in the tool banner. The Harmony tutorial prompt appears over the Budget tool.

## Evidence
```ts
// v4-layout.ts:111-118
// Mark as offered before prompting (prevents race conditions)
markTutorialOffered(tool);            // burned even if prompt is later suppressed

// Small delay to let tool render completely before showing tutorial prompt
setTimeout(() => {
  TutorialService.promptStart(tool);  // raw timer, never cleared; `tool` may be stale
}, 800);
```

## Suggested Fix
Keep the timer id at module scope and clear it at the top of `loadToolContent`; re-validate the route inside the callback:

```ts
let tutorialPromptTimer: ReturnType<typeof setTimeout> | null = null;

// in loadToolContent, before switching tools:
if (tutorialPromptTimer) { clearTimeout(tutorialPromptTimer); tutorialPromptTimer = null; }

// in promptTutorialIfFirstVisit:
tutorialPromptTimer = setTimeout(() => {
  tutorialPromptTimer = null;
  if (TOOL_TO_TUTORIAL[RouterService.getCurrentToolId()] !== tool) return; // route changed
  TutorialService.promptStart(tool);
}, 800);
```

## Why It's Hidden
Requires navigating away within an 800 ms window on a tool's very first visit — a one-shot condition per tool per profile that automated tests never hit and users rarely reproduce twice.
