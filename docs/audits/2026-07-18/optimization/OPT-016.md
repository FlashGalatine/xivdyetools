# [OPT-016]: POST /presets issues a redundant daily-limit COUNT after every successful create

## Impact
LOW

## Category
I/O (duplicate query per request)

## Location
- File: `apps/presets-api/src/handlers/presets.ts`
- Function: `presetsRouter.post('/')` — `checkSubmissionRateLimit` at line 394, then `getRemainingSubmissions` at line 504

## Current Performance
The submission handler runs `checkSubmissionRateLimit` (a `SELECT COUNT(*)` over the user's presets for today) before creating, then calls `getRemainingSubmissions` — which executes the *same* COUNT again (`rate-limit-service.ts:112-121` simply delegates to `checkSubmissionRateLimit`) — after the create, solely to populate `remaining_submissions` in the response.

## Bottleneck Analysis
One entire D1 round trip per successful submission exists only to recompute a number derivable from data already in hand: the pre-check returned `remaining`, and exactly one preset was inserted since.

## Proposed Optimization
```ts
// reuse the pre-check result instead of re-querying:
return c.json(
  {
    success: true,
    preset,
    moderation_status: status,
    remaining_submissions: Math.max(0, rateLimitResult.remaining - 1),
  },
  201
);
```

## Expected Improvement
One fewer D1 query per submission (~10-30% of the handler's D1 statements on the happy path), with correspondingly lower response latency.

## Trade-offs
The displayed number can be off by one relative to a concurrent submission from the same user — but the COUNT-based approach has the same race (BUG-049), and `remaining_submissions` is a display hint, not an enforcement value. Acceptable.

## Benchmark Approach
Compare p50 latency of `POST /api/v1/presets` (non-duplicate path) before/after in a preview deployment, or verify via D1 query analytics that per-submission statement count drops by one.

## Status

**RESOLVED 2026-07-19** — The post-create `getRemainingSubmissions` re-query was replaced by the BUG-049 enforcement count, which now also derives `remaining_submissions`; no extra statement on the happy path.
