# [OPT-009]: Bundle size — evict photon (~1.6 MiB) to a dedicated image worker; move 21 MiB of unused full-size fonts out of src/ import reach

## Impact
MEDIUM

## Category
Bundle size / deployment headroom (worker is ~8 MiB against the 10 MiB paid-plan limit)

## Location
- `apps/discord-worker/src/services/image/photon.ts:15` (top-level static import of `@cf-wasm/photon`)
- `apps/discord-worker/src/fonts/` on disk: `NotoSansKR-Variable.ttf` (10 MiB), `NotoSansSC-Regular.ttf` (11 MiB) alongside the imported `NotoSansKR-Subset.ttf` (228 KiB) and `NotoSansSC-Subset.ttf` (476 KiB) — `src/services/fonts.ts:28-40` imports only the subsets
- `apps/discord-worker/wrangler.toml:40` (`[[rules]]` treating `**/*.ttf` as Data)

## Current Performance
Of the ~8 MiB bundle, the two WASM blobs dominate: resvg (~2.4 MiB — needed by nearly every image-producing command) and photon (~1.6 MiB — needed **only** by `/match_image` and `/extractor`). Every isolate cold start pays for shipping and compiling both. Remaining headroom to the 10 MiB limit is ~2 MiB; a single dependency bump or font re-subset could consume it.

## Bottleneck Analysis
- Workers require statically-imported WASM (`photon.ts:15` — `import { PhotonImage, ... } from '@cf-wasm/photon'`), so photon cannot be lazy-loaded in-worker; it can only be *relocated*.
- The full-size source fonts in `src/fonts/` are **not** bundled today (wrangler's `[[rules]]` apply to imported modules only), but one accidental `import ... from '../fonts/NotoSansKR-Variable.ttf'` instantly adds 10 MiB and blows the limit at deploy time. Keeping 21 MiB of importable-but-unused binaries inside the glob's reach is a standing foot-gun.
- (KR-subset staleness/bloat itself was covered by the 2026-05-28 audit; not re-reported here.)

## Proposed Optimization
1. **Split image processing into a small dedicated worker** (photon + the pixel-extraction path) reached over a service binding — the pattern already used for universalis-proxy. discord-worker sends the image bytes (or the Discord CDN URL) and receives extracted palette pixels/colors.
2. **Relocate source fonts** out of `src/` (e.g. `apps/discord-worker/fonts-src/` or the scripts directory) so `[[rules]] **/*.ttf` under `src/` can only ever capture the subsets.

## Expected Improvement
~1.6 MiB (~20%) bundle reduction for discord-worker — the largest single lever short of resvg itself; faster cold starts for the 18 non-image commands; a hard guarantee against the accidental 10 MiB font import.

## Trade-offs
- One more worker to deploy/version/monitor, plus a service-binding hop per image command (negligible next to multi-second image processing).
- Passing image bytes across the binding counts against request body handling; passing the CDN URL and fetching in the image worker avoids that.
- Font relocation requires updating the subset script's output path.

## Benchmark Approach
`wrangler deploy --dry-run --outdir` before/after and compare bundle sizes; measure cold-start latency (first-request p95 after deploy) for a non-image command; verify `/match_image` and `/extractor` end-to-end latency change is within noise.
