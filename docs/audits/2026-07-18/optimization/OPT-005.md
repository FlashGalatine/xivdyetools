# [OPT-005]: og-worker swatch OG without `?sheet=` performs up to 64 sequential character-color sheet scans per request

## Impact

MEDIUM

## Category

Per-request CPU / async fan-out on an image-generation hot path

## Location

- `apps/og-worker/src/services/svg/dye-helpers.ts:70-137` (`findCharacterColorByHex`)
- Caller: `apps/og-worker/src/services/svg/swatch.ts:150-158` (runs on every `/og/swatch/:color/:limit` request lacking `?sheet=`)

## Current Performance

For every swatch OG image request without an explicit `?sheet=` parameter, `findCharacterColorByHex`:

1. Linearly scans 7 shared color categories (`getSharedColors(...).find(...)`).
2. Then **sequentially awaits** `getHairColors` / `getSkinColors` for 16 subraces × 2 genders × 2 sheets = **64 awaited lookups**, each followed by a linear `.find` over ~192 colors.

Worst case — the input color is *not* a character color, which is the common case for arbitrary swatch colors — always runs the **full** scan before returning `null`. This stacks on top of the request's resvg-wasm rasterization on a CPU-billed worker.

## Bottleneck Analysis

```ts
// dye-helpers.ts:104-134
for (const subrace of ALL_SUBRACES) {          // 16
  for (const gender of GENDERS) {              // × 2
    const hairColors = await characterColorService.getHairColors(subrace, gender);  // sequential await
    const foundHair = hairColors.find(...);
    ...
    const skinColors = await characterColorService.getSkinColors(subrace, gender);  // sequential await
    const foundSkin = skinColors.find(...);
    ...
  }
}
return null;   // ← miss pays the entire 64-step scan
```

The underlying data is static bundled JSON (loaded once per isolate by `CharacterColorService`), so the awaits resolve fast after first load — but 64 microtask round-trips plus ~12k string comparisons per request remain pure overhead, repeated on every request.

## Proposed Optimization

Precompute a reverse index once per isolate (lazy, on first swatch request):

```ts
let hexIndex: Map<string, CharacterColorLookup> | null = null;
async function getHexIndex(): Promise<Map<string, CharacterColorLookup>> {
  if (hexIndex) return hexIndex;
  const idx = new Map<string, CharacterColorLookup>();
  for (const category of sharedCategories) {
    for (const c of characterColorService.getSharedColors(category)) {
      idx.set(c.hex.toUpperCase(), { categoryName: SHARED_CATEGORY_NAMES[category], ... });
    }
  }
  for (const subrace of ALL_SUBRACES) for (const gender of GENDERS) {
    for (const c of await characterColorService.getHairColors(subrace, gender)) { idx.set(...); }
    for (const c of await characterColorService.getSkinColors(subrace, gender)) { idx.set(...); }
  }
  hexIndex = idx;
  return idx;
}
// findCharacterColorByHex → (await getHexIndex()).get(normalizedHex) ?? null
```

(Preserve current first-match precedence by only `set`ting when the key is absent.)

## Expected Improvement

- Per-request cost drops from up to 64 awaits + ~12k comparisons to one `Map.get` — O(1).
- The one-time index build costs roughly what a single miss costs today, amortized across the isolate lifetime.

## Trade-offs

- Retained memory for the map (~a few thousand entries of small objects) — trivial next to the 2.4 MiB wasm + ~770 KiB fonts already resident.
- Index must mirror the current scan's precedence order (shared categories before race-specific; iteration order preserved) to keep identical output for hex values that appear in multiple sheets.

## Benchmark Approach

`vitest` micro-benchmark: time 1,000 `findCharacterColorByHex('#123456')` (guaranteed miss) before/after; expect ~2 orders of magnitude reduction after the first call. End-to-end: `wrangler dev` + `time curl /og/swatch/A1B2C3/5.png` comparing p50 latency (wasm render will dominate; verify the delta matches the scan elimination).

## Status

**RESOLVED 2026-07-19** — `findCharacterColorByHex` reads a lazily-built reverse index (hex → lookup) instead of scanning 7 shared categories + 64 sequential awaited race/gender sheets per request; insertion order mirrors the old scan's precedence with first-write-wins, preserving identical results. Per-request cost is now one Map.get.
