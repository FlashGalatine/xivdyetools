# BUG-015: Silent Swallowing of Discord Notification Failures

## Severity
MEDIUM

## Type
Error Handling / Observability

## Location
- File: apps/presets-api/src/handlers/presets.ts
- Line(s): 479-497
- Function: POST /presets handler (notification section)

## Description
After creating a preset, the handler sends a notification to the Discord bot via service binding using `waitUntil()`. Errors are caught and logged, but there is no retry mechanism, no dead-letter queue, and no alerting. If the Discord worker is down or the service binding fails, moderators never learn about new preset submissions.

## Evidence
```typescript
// presets.ts:479-497
// PRESETS-REF-002: Fire-and-forget notification
c.executionCtx.waitUntil(
  notifyDiscordBot(c.env, {
    type: 'submission',
    preset: {
      ...preset,
      author_name: auth.userName?.trim() || 'Unknown User',
      author_discord_id: auth.userDiscordId!,
      status,
      moderation_status: moderationResult.passed ? 'clean' : 'flagged',
      source: auth.authSource,
    },
  }).catch((err) => {
    console.error(`[PRESETS-REF-002] Discord notification failed for new preset: id=${preset.id}, name="${preset.name}"`, err);
  })
);
```

## Impact
- Moderators may miss new preset submissions entirely
- No retry means the notification is permanently lost
- `console.error` only appears in worker logs, which may not be actively monitored
- The preset is still created successfully, so the user thinks everything is fine

## Suggested Fix
```typescript
// Option 1: Store failed notifications in D1 for later retry
c.executionCtx.waitUntil(
  notifyDiscordBot(c.env, notification).catch(async (err) => {
    console.error(`[PRESETS-REF-002] Discord notification failed`, err);
    // Queue for retry
    await c.env.DB.prepare(
      'INSERT INTO pending_notifications (preset_id, payload, created_at) VALUES (?, ?, ?)'
    ).bind(preset.id, JSON.stringify(notification), new Date().toISOString()).run();
  })
);

// Option 2: Simple retry with backoff
async function notifyWithRetry(env, notification, retries = 2) {
  for (let i = 0; i <= retries; i++) {
    try {
      await notifyDiscordBot(env, notification);
      return;
    } catch (err) {
      if (i === retries) console.error('Notification failed after retries', err);
      await new Promise(r => setTimeout(r, 1000 * (i + 1)));
    }
  }
}
```

## Why It's Hidden
- The preset creation succeeds regardless of notification status
- Users don't see any error
- Moderators don't know they should have been notified
- Failures only appear in worker logs (passive monitoring)
