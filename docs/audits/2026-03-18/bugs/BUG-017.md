# BUG-017: Missing Validation on Dye Array Size in Preset Submissions

## Severity
MEDIUM

## Type
Input Validation / DoS Prevention

## Location
- File: apps/presets-api/src/handlers/presets.ts
- Function: POST /presets handler (validation section)

## Description
Preset submissions accept a `dyes` array in the request body, but there is no explicit validation on the array size. A malicious client could submit a preset with 0 dyes (meaningless) or 10,000+ dyes (resource exhaustion during duplicate signature generation, JSON serialization, moderation, and database storage).

## Evidence
The preset creation handler validates individual dye entries but not the array bounds:
```typescript
// Missing bounds check on body.dyes
// body.dyes is used to:
// 1. Generate dye_signature via generateDyeSignature() — sorts + JSON.stringify
// 2. Store as JSON in D1 column
// 3. Pass to moderation pipeline
// 4. Return in API response
```

The `generateDyeSignature()` function in `preset-service.ts:28-31` sorts and stringifies the array:
```typescript
export function generateDyeSignature(dyes: number[]): string {
  const sorted = [...dyes].sort((a, b) => a - b);  // O(n log n) for large arrays
  return JSON.stringify(sorted);
}
```

## Impact
- **0 dyes**: Empty preset with no visual content — clutters the database
- **1000+ dyes**: Sort + stringify becomes expensive; D1 column size limit may be hit
- **10,000+ dyes**: Could cause Worker CPU time limit exceeded (30 seconds on paid plan)
- The `dye_signature` column with excessive data may degrade D1 query performance

## Suggested Fix
```typescript
// In validation section of POST /presets handler:
if (!Array.isArray(body.dyes) || body.dyes.length < 1 || body.dyes.length > 200) {
  return c.json({
    error: 'Preset must contain between 1 and 200 dyes',
    provided: Array.isArray(body.dyes) ? body.dyes.length : 0,
  }, 400);
}
```

The upper bound of 200 is generous — FFXIV gear has ~15 dyeable slots maximum, so realistic presets would have 1-15 dyes.

## Why It's Hidden
- All legitimate presets from the web app and Discord bot have reasonable dye counts
- The attack requires crafting a raw API request
- No existing tests cover boundary conditions for array size
