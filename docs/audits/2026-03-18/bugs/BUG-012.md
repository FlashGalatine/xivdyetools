# BUG-012: Unsafe JSON.parse() Without Try-Catch on D1 JSON Columns

## Severity
HIGH

## Type
Error Handling / Data Integrity

## Location
- File: apps/presets-api/src/services/preset-service.ts
- Line(s): 42-43, 52
- Function: rowToPreset()

## Description
The `rowToPreset()` function calls `JSON.parse()` on three D1 database columns (`dyes`, `tags`, `previous_values`) without try-catch error handling. If any of these columns contains malformed JSON — due to database corruption, manual SQL edits, migration errors, or edge cases in D1's SQLite implementation — the entire request crashes with an unhandled exception, returning a 500 error to the client.

## Evidence
```typescript
// preset-service.ts:36-54
export function rowToPreset(row: PresetRow): CommunityPreset {
  return {
    id: row.id,
    name: row.name,
    description: row.description,
    category_id: row.category_id as CommunityPreset['category_id'],
    dyes: JSON.parse(row.dyes) as CommunityPreset['dyes'],          // No try-catch
    tags: JSON.parse(row.tags) as CommunityPreset['tags'],          // No try-catch
    author_discord_id: row.author_discord_id,
    author_name: row.author_name,
    vote_count: row.vote_count,
    status: row.status as CommunityPreset['status'],
    is_curated: row.is_curated === 1,
    created_at: row.created_at,
    updated_at: row.updated_at,
    dye_signature: row.dye_signature || undefined,
    previous_values: row.previous_values
      ? (JSON.parse(row.previous_values) as CommunityPreset['previous_values'])  // No try-catch
      : null,
  };
}
```

## Reproduction Scenario
1. A D1 migration partially fails, leaving a row with truncated JSON in the `dyes` column
2. User browses `/api/v1/presets` which queries multiple presets
3. `rowToPreset()` is called for each row
4. One malformed row causes `JSON.parse()` to throw `SyntaxError`
5. **All presets fail to load** — not just the corrupted one

## Impact
One corrupted row can take down the entire presets listing endpoint. This is a **single point of failure** where bad data in one row cascades to affect all users.

## Suggested Fix
```typescript
function safeJsonParse<T>(json: string, fallback: T): T {
  try {
    return JSON.parse(json) as T;
  } catch {
    console.error(`[PRESET-DATA] Malformed JSON in database: ${json.slice(0, 100)}`);
    return fallback;
  }
}

export function rowToPreset(row: PresetRow): CommunityPreset {
  return {
    // ...
    dyes: safeJsonParse<CommunityPreset['dyes']>(row.dyes, []),
    tags: safeJsonParse<CommunityPreset['tags']>(row.tags, []),
    previous_values: row.previous_values
      ? safeJsonParse<CommunityPreset['previous_values']>(row.previous_values, null)
      : null,
  };
}
```

## Why It's Hidden
- D1 data is always written via validated code paths (`JSON.stringify()` in `createPreset()`)
- In normal operation, the JSON is always valid
- Only manifests from data corruption, which is rare but catastrophic when it occurs
- No test covers the malformed JSON scenario
