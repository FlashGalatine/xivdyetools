# OPT-001: Category Cache Thundering Herd on Expiration

## Impact
MEDIUM

## Category
Caching / I/O Optimization

## Location
- File: apps/presets-api/src/handlers/presets.ts
- Line(s): 517-549
- Function: getValidCategories()

## Current Performance
Category validation queries D1 for valid category IDs. The results are cached at module level with a 60-second TTL. When the cache expires, the next request triggers a fresh D1 query. However, if multiple concurrent requests arrive during cache expiration, they ALL miss the cache and ALL query D1 simultaneously.

## Bottleneck Analysis
```typescript
// presets.ts:517-549
let cachedCategories: string[] | null = null;
let categoryCacheTime = 0;

async function getValidCategories(db: D1Database): Promise<string[]> {
  const now = Date.now();
  if (cachedCategories && now - categoryCacheTime < CATEGORY_CACHE_TTL) {
    return cachedCategories;
  }
  // No deduplication — all concurrent requests reach this point
  const result = await db.prepare('SELECT id FROM categories').all<{ id: string }>();
  cachedCategories = result.results.map((r) => r.id);
  categoryCacheTime = now;
  return cachedCategories;
}
```

Under high traffic (e.g., 50 concurrent preset submissions right after cache expires), all 50 requests execute the same `SELECT id FROM categories` query.

## Proposed Optimization
Use a pending promise pattern (same approach as APIService.getPriceData):
```typescript
let cachedCategories: string[] | null = null;
let categoryCacheTime = 0;
let pendingCategoryFetch: Promise<string[]> | null = null;

async function getValidCategories(db: D1Database): Promise<string[]> {
  const now = Date.now();
  if (cachedCategories && now - categoryCacheTime < CATEGORY_CACHE_TTL) {
    return cachedCategories;
  }

  // Deduplicate: if a fetch is already in progress, wait for it
  if (pendingCategoryFetch) return pendingCategoryFetch;

  pendingCategoryFetch = (async () => {
    try {
      const result = await db.prepare('SELECT id FROM categories').all<{ id: string }>();
      cachedCategories = result.results.map((r) => r.id);
      categoryCacheTime = Date.now();
      return cachedCategories;
    } finally {
      pendingCategoryFetch = null;
    }
  })();

  return pendingCategoryFetch;
}
```

## Expected Improvement
- D1 queries reduced from N (concurrent requests) to 1 per cache miss
- Response time unchanged for the first request; subsequent requests resolve faster
- D1 read load reduced during traffic spikes

## Trade-offs
- Slightly more complex code
- If the single query fails, all waiting requests fail (but this is already the case for each individual query)

## Benchmark Approach
- Log D1 query count per minute via Worker analytics
- Compare before/after during peak traffic periods
