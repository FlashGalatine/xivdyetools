# DEAD-053: Tracked `output/dye_names.csv` despite `.gitignore` rule

## Category
Orphaned Files

## Location
- File(s): `packages/core/scripts/output/dye_names.csv` (137 lines)
- Related: `packages/core/scripts/.gitignore`

## Evidence
The `scripts/.gitignore` contains:
```
# Script outputs
output/*.csv
```

Yet `output/dye_names.csv` (137 lines) is tracked in git. This file is the output of `fetch_dye_names.py` — it contains dye names in multiple languages fetched from XIVAPI.

The file is not imported by any script or build process. The build-locales pipeline (`build-locales.ts`) reads from the top-level `dyenames.csv` (in the core package root), not from `scripts/output/dye_names.csv`.

Also, `scripts/output/.gitkeep` exists alongside the CSV, which is contradictory — `.gitkeep` is meant to preserve empty directories, but the CSV is tracked despite the gitignore rule.

## Why It Exists
Output artifact from `fetch_dye_names.py` that was committed before the `.gitignore` rule was added (git continues tracking files that were committed before being gitignored).

## Removal Risk Assessment

| Factor | Assessment |
|--------|------------|
| **Confidence** | HIGH — gitignored output artifact |
| **Blast Radius** | NONE |
| **Reversibility** | EASY — re-run `fetch_dye_names.py` to regenerate |
| **Hidden Consumers** | None |

## Recommendation
**REMOVE**

### Rationale
Generated output that should have been gitignored. Can be regenerated by running `fetch_dye_names.py`.

### If Removing
1. `git rm --cached scripts/output/dye_names.csv`
2. Verify `.gitignore` rule `output/*.csv` takes effect
3. Optionally remove `scripts/output/.gitkeep` and add a note in README.md
