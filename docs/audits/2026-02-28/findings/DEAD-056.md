# DEAD-056: `VERSION` export — zero external consumers

## Category
Unused Exports

## Location
- File(s): `packages/core/src/version.ts`, `packages/core/src/index.ts` (line 170)
- Symbol(s): `VERSION`

## Evidence
`VERSION` is a string constant auto-generated by `scripts/generate-version.ts` from `package.json` during builds:

```typescript
export const VERSION = '1.17.3';
```

**External consumer usage:** Zero. No monorepo project imports `VERSION` from `@xivdyetools/core`.

**Internal usage:** Not used by any service. It exists purely as a public API for consumers to check which core version they're running.

## Why It Exists
Standard practice for npm packages to expose their version at runtime. Useful for diagnostics, user-agent strings, and "About" screens.

## Removal Risk Assessment

| Factor | Assessment |
|--------|------------|
| **Confidence** | LOW — standard npm pattern; legitimate use case even without current consumers |
| **Blast Radius** | NONE |
| **Reversibility** | EASY |
| **Hidden Consumers** | External npm consumers likely use this for diagnostics |

## Recommendation
**KEEP**

### Rationale
`VERSION` is a standard npm library pattern (10 bytes overhead). Removing it would be unusual and unhelpful. The build script that generates it is already part of the build pipeline.
