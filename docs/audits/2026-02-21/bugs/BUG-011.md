# BUG-011: `DelegatingLogger.time()` Loses Child Context

## Severity
LOW

## Type
Logic Error

## Location
- File: `packages/logger/src/core/base-logger.ts`
- Line(s): ~308-310
- Function: `DelegatingLogger.time()`

## Description
When a child logger calls `time()`, the resulting debug log goes through the parent directly, bypassing the child's context merge. The timing log will miss the child's `requestId`, `operation`, and other context fields.

## Evidence
```typescript
time(label: string): () => number {
  return this.parent.time(label);  // â† Delegates to parent, skips child context
}
```

## Suggested Fix
Override `time()` to use the child's own logging:

```typescript
time(label: string): () => number {
  const start = typeof performance !== 'undefined' ? performance.now() : Date.now();
  return () => {
    const end = typeof performance !== 'undefined' ? performance.now() : Date.now();
    const duration = end - start;
    this.debug(`${label}: ${duration.toFixed(2)}ms`, { duration, label });
    return duration;
  };
}
```

## Why It's Hidden
Timing debug logs are low-volume and often not inspected closely. The missing context fields would only be noticed when correlating timing logs with request-scoped data in structured log querying.
