# BUG-007: D1 Mock `batch()` Discards Statement Results

## Severity
MEDIUM

## Type
Behavioral Discrepancy (Mock vs Production)

## Location
- File: `packages/test-utils/src/cloudflare/d1.ts`
- Line(s): ~278-286
- Function: `batch()`

## Description
The D1 mock's `batch()` implementation calls `stmt.run()` for each statement but discards the return value, always returning empty results. Real D1 `batch()` returns the results of each statement.

## Evidence
```typescript
batch: async <T = unknown>(statements: MockD1PreparedStatement[]): Promise<D1Result<T>[]> => {
  const results: D1Result<T>[] = [];
  for (const stmt of statements) {
    await stmt.run();  // ‚Üê run() result is DISCARDED
    results.push({ results: [] as T[], success: true as const, meta: createDefaultMeta() });
  }
  return results;
},
```

## Reproduction Scenario
Any test using `batch()` to verify returned data from multiple queries will always see empty result sets, even when mock functions are configured to return data.

## Suggested Fix
```typescript
batch: async <T = unknown>(statements: MockD1PreparedStatement[]): Promise<D1Result<T>[]> => {
  const results: D1Result<T>[] = [];
  for (const stmt of statements) {
    const result = await stmt.run<T>();
    results.push(result);
  }
  return results;
},
```

## Why It's Hidden
Tests that use `batch()` may only be testing for `success: true` without inspecting the result data, or they may use individual statement execution instead.
