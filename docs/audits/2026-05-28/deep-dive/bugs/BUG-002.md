# [BUG-002]: og-worker casts string enum route/query params without membership validation

## Severity
LOW (robustness / wasted compute — no crash, no security impact)

## Type
Input validation gap

## Location
- File: `apps/og-worker/src/index.ts`
- Examples: harmony (208-211), mixer/gradient/accessibility handlers, swatch
- Downstream: `services/svg/*.ts`

## Description
The image routes carefully NaN-guard their **numeric** params (FINDING-011, e.g.
`if (isNaN(dyeId)) return 400`), but their **string enum** params are type-*asserted* rather
than validated:

```typescript
const harmonyType = harmonyTypeRaw.toLowerCase() as HarmonyType;            // any string
const algorithm = (c.req.query('algo') || 'oklab') as MatchingAlgorithm;    // any string
```

`as HarmonyType` / `as MatchingAlgorithm` are compile-time-only casts; at runtime the value can
be any attacker-controlled string. The same applies to `visionType` in the accessibility route.

## Impact
Bounded and non-fatal. Tracing the harmony path:
- `getHarmonyMatches` has a `switch (harmonyType)` with a `default:` branch ([harmony.ts:105](../../../apps/og-worker/src/services/svg/harmony.ts#L105)), so an unknown type degrades to the default rather than throwing.
- The card subtitle is `harmonyNames[harmonyType] || harmonyType`, so a bogus value is echoed verbatim into the rendered PNG.

So `GET /og/harmony/5771/banana.png` returns **200** with a degraded card whose subtitle reads
"banana", and still pays the full resvg rasterization cost. No crash, no injection (output is
XML-escaped), but: (a) wasted CPU on garbage input, (b) cache pollution with junk keys, (c)
inconsistent contract vs the numeric params which *are* rejected with 400.

## Why It's Hidden
Tests pass valid enum values; TypeScript's `as` cast suppresses the very type error that would
otherwise force validation; and the graceful `default:` branches mean nothing visibly breaks.

## Suggested Fix
Validate enum params against their known sets and return 400 on miss, mirroring the numeric
guards. Reuse the existing constant arrays (`SUPPORTED_TOOLS`-style):

```typescript
const HARMONY_TYPES = ['complementary','analogous','triadic','split-complementary',
  'tetradic','square','monochromatic','compound','shades'] as const;
const ALGORITHMS = ['oklab','ciede2000','euclidean'] as const;

if (!HARMONY_TYPES.includes(harmonyTypeRaw as HarmonyType)) {
  return c.json({ error: 'Invalid harmony type' }, 400);
}
if (!ALGORITHMS.includes(algorithm)) {
  return c.json({ error: 'Invalid algorithm' }, 400);
}
```

This rejects junk before the expensive render and keeps the cache clean.
