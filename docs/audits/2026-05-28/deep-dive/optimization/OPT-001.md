# [OPT-001]: KR subset font carries ~595 KiB of unused CJK glyphs

## Impact
HIGH (bundle size — meaningful headroom against the 10 MiB Worker limit)

## Category
Memory / Bundle size

## Location
- Asset: `apps/discord-worker/src/fonts/NotoSansKR-Subset.ttf`
- Producer: `apps/discord-worker/scripts/subset-cjk-fonts.py:193`

## Current Performance
`NotoSansKR-Subset.ttf` is **801–820 KiB** and its cmap contains **821 CJK Han ideographs** plus
459 Hangul syllables. Korean locale text uses **zero** CJK ideographs (verified: ko.json contains
only Hangul). Those 821 Han glyphs are never rendered — the runtime font stack
(`getFontWithCjkFallback`) routes CJK to Noto Sans SC.

## Bottleneck Analysis
`subset-cjk-fonts.py` subsets **both** SC and KR with the *same* `codepoints` set — the union of
all characters from all locales, including every ja/zh CJK ideograph:

```python
kr_size, kr_glyphs = subset_font(kr_input, KR_OUTPUT, codepoints, fix_names={...})  # line 193
```

Noto Sans KR ships Hanja, so it greedily retains every requested CJK glyph. Noto Sans SC avoids
the mirror-image problem only because it has no Hangul glyphs to pull in.

## Proposed Optimization
Scope the KR subsetter to Hangul + ASCII only:

```python
HANGUL = lambda c: 0xAC00 <= c <= 0xD7AF or 0x1100 <= c <= 0x11FF
kr_codepoints = {c for c in codepoints if c < 0x80 or HANGUL(c)}
kr_size, kr_glyphs = subset_font(kr_input, KR_OUTPUT, kr_codepoints, fix_names={...})
```

## Expected Improvement
Empirically verified by re-subsetting the source font to Hangul + ASCII only:

| | Current | Optimized |
|---|---------|-----------|
| NotoSansKR-Subset.ttf | ~801 KiB | **~225 KiB** |
| Savings | — | **~595 KiB** (~74% smaller) |

The discord-worker bundle is ~8 MiB against a 10 MiB limit (project memory), so ~0.6 MiB of
recovered headroom is material.

## Trade-offs
None functional — the removed glyphs are never used. The only caveat: if a future Korean string
ever needs Hanja, the filter would need to include those specific codepoints (none today).

## Benchmark Approach
Re-run the patched script; `ls -l NotoSansKR-Subset.ttf`; confirm the discord-worker still renders
Korean dye names correctly (`/swatch` with `?lang=ko`); confirm `wrangler deploy` reports a smaller
upload. Pair with the staleness re-subset (covers the 2 currently-missing Hangul glyphs 믹/빌 from
the i18n audit) in the same commit.
